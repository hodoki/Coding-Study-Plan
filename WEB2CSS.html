CSS의 등장
css가 이것을 가능하게 해줌 style태그 안에 있는 코드는 html이 아닌 css로 읽으라고 웹 브라우저에게 명령함 그리고 단순한 코드 하나를 변경함으로써 1억개의 글자를 다 바꿀 수 있게 해줌 ㅅ발 혁신이였던거임 그리고 css는 첫째, html이 너무나 중요하기 떄문에(가장 중요한 정보 전달의 역할을 하기 때문-시각 장애인은 디자인은 못보지만 적혀있는 내용은 볼 수 있다.) html이 정보에 전념하게 하기 위해서 html로부터 디자인에 대한 기능을 뺏어왔다. 둘째, css를 통해서 디자인 하는 것이 html을 통해서 디자인 하는 것 보다 훨씬 더 효율적이기 때문에 css가 도입되었다.

세미콜론은 디스크립션의 시작과 끝을 구분한다.
하나의 선택자에 대해서 여러 개의 효과를 지정할 수 있는데 그 효과들을 세미콜론으로 구분한다.
a {
(property)color:(value)black;
text-decoration:none;
}

CSS 속성을 스스로 알아내는 방법

text-decoration:none;
}

CSS 선택자를 스스로 알아내는 방법.
선택자끼리의 우선순위가 동일하면 가장 최근에 명령한(가장 마지막에 등장하는) 선택자가 이긴다.
그러나 id선택자와 class선택자가 붙으면 id선택자가 이긴다.
class선택자와 태그 선택자가 붙으며 class선택자가 이긴다.
class에 해당하는 class선택자는 앞에 .을 붙인다.
id에 해당하는 id선택자는 앞에 #을 붙인다.
원리:구체적인 것이 포괄적인 것보다 우선순위가 높다.
먼저 전체적인 디자인을 태그 선택자를 통해서 해놓고 그 다음 구체적인 것들을 설정할 때 id선택자를 통해서 예외를 두어 선택해 나갈 수 있기 때문이다.
클래스 선택자는 태그 선택자 보다는 우서순위가 높지만 id선택자 보다는 우선순위가 낮다.
